    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-angular-seed--the-seed-for-angularjs-apps" class="anchor" href="#angular-seed--the-seed-for-angularjs-apps" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>angular-seed</code> — the seed for AngularJS apps</h1>
<p>This project is an application skeleton for a typical <a href="https://angularjs.org/">AngularJS</a> web app. You can use it
to quickly bootstrap your angular webapp projects and dev environment for these projects.</p>
<p>The seed contains a sample AngularJS application and is preconfigured to install the Angular
framework and a bunch of development and testing tools for instant web development gratification.</p>
<p>The seed app doesn't do much, just shows how to wire two controllers and views together.</p>
<h2><a id="user-content-getting-started" class="anchor" href="#getting-started" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Getting Started</h2>
<p>To get you started you can simply clone the <code>angular-seed</code> repository and install the dependencies:</p>
<h3><a id="user-content-prerequisites" class="anchor" href="#prerequisites" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Prerequisites</h3>
<p>You need git to clone the <code>angular-seed</code> repository. You can get git from <a href="https://git-scm.com/">here</a>.</p>
<p>We also use a number of Node.js tools to initialize and test <code>angular-seed</code>. You must have Node.js
and its package manager (npm) installed. You can get them from <a href="https://nodejs.org/">here</a>.</p>
<h3><a id="user-content-clone-angular-seed" class="anchor" href="#clone-angular-seed" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Clone <code>angular-seed</code></h3>
<p>Clone the <code>angular-seed</code> repository using git:</p>
<pre><code>git clone https://github.com/angular/angular-seed.git
cd angular-seed
</code></pre>
<p>If you just want to start a new project without the <code>angular-seed</code> commit history then you can do:</p>
<pre><code>git clone --depth=1 https://github.com/angular/angular-seed.git &lt;your-project-name&gt;
</code></pre>
<p>The <code>depth=1</code> tells git to only pull down one commit worth of historical data.</p>
<h3><a id="user-content-install-dependencies" class="anchor" href="#install-dependencies" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Install Dependencies</h3>
<p>We have two kinds of dependencies in this project: tools and Angular framework code. The tools help
us manage and test the application.</p>
<ul>
<li>We get the tools we depend upon via <code>npm</code>, the <a href="https://www.npmjs.org/">Node package manager</a>.</li>
<li>We get the Angular code via <code>bower</code>, a <a href="http://bower.io/">client-side code package manager</a>.</li>
<li>In order to run the end-to-end tests, you will also need to have the
<a href="https://wikipedia.org/wiki/Java_Development_Kit">Java Development Kit (JDK)</a> installed on your machine. Check out the section on
<a href="#e2e-testing">end-to-end testing</a> for more info.</li>
</ul>
<p>We have preconfigured <code>npm</code> to automatically run <code>bower</code> so we can simply do:</p>
<pre><code>npm install
</code></pre>
<p>Behind the scenes this will also call <code>bower install</code>. After that, you should find out that you have
two new folders in your project.</p>
<ul>
<li><code>node_modules</code> - contains the npm packages for the tools we need</li>
<li><code>app/bower_components</code> - contains the Angular framework files</li>
</ul>
<p><em>Note that the <code>bower_components</code> folder would normally be installed in the root folder but
<code>angular-seed</code> changes this location through the <code>.bowerrc</code> file. Putting it in the <code>app</code> folder
makes it easier to serve the files by a web server.</em></p>
<h3><a id="user-content-run-the-application" class="anchor" href="#run-the-application" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Run the Application</h3>
<p>We have preconfigured the project with a simple development web server. The simplest way to start
this server is:</p>
<pre><code>npm start
</code></pre>
<p>Now browse to the app at <a href="http://localhost:8000/index.html"><code>localhost:8000/index.html</code></a>.</p>
<h2><a id="user-content-directory-layout" class="anchor" href="#directory-layout" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Directory Layout</h2>

<h2><a id="user-content-testing" class="anchor" href="#testing" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Testing</h2>
<p>There are two kinds of tests in the <code>angular-seed</code> application: Unit tests and end-to-end tests.</p>
<h3><a id="user-content-running-unit-tests" class="anchor" href="#running-unit-tests" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running Unit Tests</h3>
<p>The <code>angular-seed</code> app comes preconfigured with unit tests. These are written in <a href="https://jasmine.github.io/">Jasmine</a>,
which we run with the <a href="https://karma-runner.github.io/">Karma</a> test runner. We provide a Karma configuration file to run them.</p>
<ul>
<li>The configuration is found at <code>karma.conf.js</code>.</li>
<li>The unit tests are found next to the code they are testing and have an <code>_test.js</code> suffix (e.g.
<code>view1_test.js</code>).</li>
</ul>
<p>The easiest way to run the unit tests is to use the supplied npm script:</p>
<pre><code>npm test
</code></pre>
<p>This script will start the Karma test runner to execute the unit tests. Moreover, Karma will start
watching the source and test files for changes and then re-run the tests whenever any of them
changes.
This is the recommended strategy; if your unit tests are being run every time you save a file then
you receive instant feedback on any changes that break the expected code functionality.</p>
<p>You can also ask Karma to do a single run of the tests and then exit. This is useful if you want to
check that a particular version of the code is operating as expected. The project contains a
predefined script to do this:</p>
<pre><code>npm run test-single-run
</code></pre>
<p><a name="user-content-e2e-testing"></a></p>
<h3><a id="user-content-running-end-to-end-tests" class="anchor" href="#running-end-to-end-tests" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running End-to-End Tests</h3>
<p>The <code>angular-seed</code> app comes with end-to-end tests, again written in <a href="https://jasmine.github.io/">Jasmine</a>. These tests
are run with the <a href="http://www.protractortest.org/">Protractor</a> End-to-End test runner. It uses native events and has
special features for Angular applications.</p>
<ul>
<li>The configuration is found at <code>e2e-tests/protractor-conf.js</code>.</li>
<li>The end-to-end tests are found in <code>e2e-tests/scenarios.js</code>.</li>
</ul>
<p>Protractor simulates interaction with our web app and verifies that the application responds
correctly. Therefore, our web server needs to be serving up the application, so that Protractor can
interact with it.</p>
<p><strong>Before starting Protractor, open a separate terminal window and run:</strong></p>
<pre><code>npm start
</code></pre>
<p>In addition, since Protractor is built upon WebDriver, we need to ensure that it is installed and
up-to-date. The <code>angular-seed</code> project is configured to do this automatically before running the
end-to-end tests, so you don't need to worry about it. If you want to manually update the WebDriver,
you can run:</p>
<pre><code>npm run update-webdriver
</code></pre>
<p>Once you have ensured that the development web server hosting our application is up and running, you
can run the end-to-end tests using the supplied npm script:</p>
<pre><code>npm run protractor
</code></pre>
<p>This script will execute the end-to-end tests against the application being hosted on the
development server.</p>
<p><strong>Note:</strong>
Under the hood, Protractor uses the <a href="http://docs.seleniumhq.org/">Selenium Standalone Server</a>, which in turn requires
the <a href="https://wikipedia.org/wiki/Java_Development_Kit">Java Development Kit (JDK)</a> to be installed on your local machine. Check this by running
<code>java -version</code> from the command line.</p>
<p>If JDK is not already installed, you can download it <a href="http://www.oracle.com/technetwork/java/javase/downloads">here</a>.</p>
<h2><a id="user-content-updating-angular" class="anchor" href="#updating-angular" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Updating Angular</h2>
<p>Since the Angular framework library code and tools are acquired through package managers (npm and
bower) you can use these tools to easily update the dependencies. Simply run the preconfigured
script:</p>
<pre><code>npm run update-deps
</code></pre>
<p>This will call <code>npm update</code> and <code>bower update</code>, which in turn will find and install the latest
versions that match the version ranges specified in the <code>package.json</code> and <code>bower.json</code> files
respectively.</p>
<h2><a id="user-content-loading-angular-asynchronously" class="anchor" href="#loading-angular-asynchronously" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Loading Angular Asynchronously</h2>
<p>The <code>angular-seed</code> project supports loading the framework and application scripts asynchronously.
The special <code>index-async.html</code> is designed to support this style of loading. For it to work you must
inject a piece of Angular JavaScript into the HTML page. The project has a predefined script to help
do this:</p>
<pre><code>npm run update-index-async
</code></pre>
<p>This will copy the contents of the <code>angular-loader.js</code> library file into the <code>index-async.html</code>
page. You can run this every time you update the version of Angular that you are using.</p>
<h2><a id="user-content-serving-the-application-files" class="anchor" href="#serving-the-application-files" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Serving the Application Files</h2>
<p>While Angular is client-side-only technology and it is possible to create Angular web apps that
do not require a backend server at all, we recommend serving the project files using a local
web server during development to avoid issues with security restrictions (sandbox) in browsers. The
sandbox implementation varies between browsers, but quite often prevents things like cookies, XHR,
etc to function properly when an HTML page is opened via the <code>file://</code> scheme instead of <code>http://</code>.</p>
<h3><a id="user-content-running-the-app-during-development" class="anchor" href="#running-the-app-during-development" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running the App during Development</h3>
<p>The <code>angular-seed</code> project comes preconfigured with a local development web server. It is a Node.js
tool called <a href="https://github.com/indexzero/http-server">http-server</a>. You can start this web server with <code>npm start</code>, but you may
choose to install the tool globally:</p>
<pre><code>sudo npm install -g http-server
</code></pre>
<p>Then you can start your own development web server to serve static files from a folder by running:</p>
<pre><code>http-server -a localhost -p 8000
</code></pre>
<p>Alternatively, you can choose to configure your own web server, such as Apache or Nginx. Just
configure your server to serve the files under the <code>app/</code> directory.</p>
<h3><a id="user-content-running-the-app-in-production" class="anchor" href="#running-the-app-in-production" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running the App in Production</h3>
<p>This really depends on how complex your app is and the overall infrastructure of your system, but
the general rule is that all you need in production are the files under the <code>app/</code> directory.
Everything else should be omitted.</p>
<p>Angular apps are really just a bunch of static HTML, CSS and JavaScript files that need to be hosted
somewhere they can be accessed by browsers.</p>
<p>If your Angular app is talking to the backend server via XHR or other means, you need to figure out
what is the best way to host the static files to comply with the same origin policy if applicable.
Usually this is done by hosting the files by the backend server or through reverse-proxying the
backend server(s) and web server(s).</p>
<h2><a id="user-content-continuous-integration" class="anchor" href="#continuous-integration" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Continuous Integration</h2>
<h3><a id="user-content-travis-ci" class="anchor" href="#travis-ci" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Travis CI</h3>
<p><a href="https://travis-ci.org/">Travis CI</a> is a continuous integration service, which can monitor GitHub for new commits to
your repository and execute scripts such as building the app or running tests. The <code>angular-seed</code>
project contains a Travis configuration file, <code>.travis.yml</code>, which will cause Travis to run your
tests when you push to GitHub.</p>
<p>You will need to enable the integration between Travis and GitHub. See the
<a href="https://docs.travis-ci.com/user/getting-started">Travis website</a> for instructions on how to do this.</p>
